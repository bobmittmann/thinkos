\section{Code conventions}

Commonly, small embedded RTOS kernels are linked against the user application, and consequently sharing the same address and name spaces.

Internal global functions, used by the several thinkos modules, are prefixed by \var(__thinkos_). There is no convention for static functions inside a single compilation unit.

Service definitions have the suffix \var(_svc).

Thread level user calls, either service or library, are be prefixed by \var(thinkos_) 

Interrupt level user calls, are be prefixed by \var(thinkos_) and suffixed by \var(_i)
